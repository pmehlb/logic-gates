with that being said, here's a key of all the logic gates i'll be using


note: i'll be using a LOT of pseudo-code

NOT Gate: inverts the input (on goes to off, off goes to on)


(pseudo)code example: 
if input is on:
	turn output off
if input is off:
	turn output on

OR Gate: if either or both inputs are on, output is on


(pseudo)code example: 
if input A is on:
	turn output on
if input B is on:
	turn output on
if inputs A and B are on:
	turn output on

AND Gate: if both inputs are on, output is on


(pseudo)code example: 
if input A is on:
	turn output off
if input B is on:
	turn output off
if inputs A and B are on:
	turn output on
